---
title: Guidelines
---

Just a few things to keep in mind when contributing to Mist.

- Be respectful and considerate of others, especially when providing feedback.
- All commits should follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) standard. To make this easy,
  [cocogitto](https://docs.cocogitto.io) is included in the project.
- All commits should be [signed](https://docs.github.com/en/github/authenticating-to-github/signing-commits).
- Any new features should have corresponding tests.
- Please open an [Issue](https://github.com/mist-id/mist/issues) before submitting a PR to discuss the proposed changes, especially for major changes.
